User Journey: Meeting Co-Pilot

1. Pre-Meeting / Onboarding
Goal: Prepare for a new meeting or access past information.

* Landing Page: The user arrives at the home page (`/`), which serves as the main dashboard.
* Navigation: A collapsible sidebar on the left allows quick access to:
    * Home: The main recording interface.
    * Meeting Notes: A folder view of all past meetings, organized hierarchically.
    * Settings: Configuration for AI models (Gemini, Ollama, Groq), transcription settings, and API keys.
    * Import: A dedicated button to upload existing audio files for processing.
    * Search: A global search bar to find past meetings by title or transcript content.

2. During the Meeting (Real-Time Experience)
Goal: Capture the conversation and get live assistance.

* Starting a Recording:
    * The user clicks the large Microphone button on the home page.
    * Real-Time Transcription: As participants speak, the text appears immediately on the screen. The system uses a WebSocket connection (`/ws/streaming-audio`) to stream audio to a Groq-powered Whisper model for low-latency transcription.
    * Visual Feedback: An audio visualizer (bars) reacts to voice activity, confirming the microphone is working.
* Meeting Controls:
    * Pause/Resume: The user can pause the recording during breaks and resume seamlessly without creating a new file.
    * Stop: Clicking the "Stop" button ends the session.
* Live Updates: The transcript updates in real-time, handling "partial" results (gray text) that stabilize into "final" results (black text) as the model confirms the sentence structure.

3. Post-Meeting (Review & Analysis)
Goal: Digest the meeting content and extract actionable insights.

* Automatic Saving: Upon stopping the recording, the transcript is automatically saved to the database (SQLite). The user is redirected to the Meeting Details page (`/meeting-details?id=...`).
* AI Summary Generation:
    * The system automatically triggers an AI process (using models like Gemini Flash) to generate a structured summary.
    * Structured Output: The user sees a formatted report containing:
        * Key Points: High-level takeaways.
        * Action Items: Specific tasks assigned to people.
        * Decisions: tough choices made during the call.
* Transcript Review:
    * The full transcript is available for reading.
    * Diarization: The user can trigger "Diarization" to identify distinct speakers (Speaker A, Speaker B).
    * Editing: The meeting title can be renamed (e.g., from "Meeting 25_02_2026" to "Q1 Planning").

4. Interactive Intelligence (Chat & Context)
Goal: Query the meeting data for specific answers.

* Ask AI (Chat Interface):
    * A floating "Ask AI" button opens a chat panel on the right.
    * Context-Aware Q&A: The user can ask questions like "What was the budget decision?" The AI answers based only on the current meeting's transcript.
    * Web Search: The user can ask the bot to "search on web [query]" to fact-check or gather external info during the review.
* Cross-Meeting Context:
    * Linking: The user can "Link Context" by selecting other past meetings from a dropdown.
    * Synthesized Answers: The AI can now answer questions that span multiple meetings (e.g., "How has the project scope changed since last week's call?").

5. Management & Organization
Goal: Maintain a knowledge base.

* Search & Retrieval: Using the sidebar search, the user can type a keyword (e.g., "budget") and instantly see which meetings contain that term, with specific transcript snippets highlighted.
* Importing: If the user recorded a meeting on another device, they can use the Import feature to upload the audio file. The system processes it through the same transcription -> summarization pipeline.
* Export: The user can copy the summary or transcript to their clipboard to paste into email or tools like Slack/Jira.
